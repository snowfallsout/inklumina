# DEVNOTES_2026-05-02T210000Z

## 背景

在 `function/runtime` 收斂為 kernel-only 之後，`src/lib/function` 根目錄還留下：

- `camAndMedia.js`
- `particleFull.js`
- `_CORE_FILES_.md`

其中：

- `camAndMedia.js` 已無任何活躍引用
- `particleFull.js` 仍可作為 monolith 行為參考
- `_CORE_FILES_.md` 內容已嚴重落後於目前 service-centric 結構

## 本輪決策

### A. 刪除 `camAndMedia.js`

刪除理由：

1. workspace 中沒有任何活躍 import / script 依賴
2. 與目前 `services/display/camera.ts`、`services/display/runtime.ts` 已無關聯
3. 保留它只會延續舊 JS runtime 的錯誤印象

雖然 `particleFull.js` 也沒有活躍引用，但它仍是較完整的 legacy monolith，可在必要時作為行為參考；本輪先不刪。

### C. 更新 `_CORE_FILES_.md`

將 `_CORE_FILES_.md` 改寫為目前實況：

- `function/runtime/*` 僅剩 kernel
- `services/display/*` 已接手 bootstrap / realtime / camera / legacy / types owner
- `particleFull.js` 為保留中的歷史參考檔

## 本輪不做

本輪刻意不做以下事項：

- 不刪除 `particleFull.js`
- 不重命名 `function/runtime`
- 不改動任何 active runtime 行為

## 驗證方式

本輪完成後需確認：

- `npm run check` 通過
- workspace 中不再有 `camAndMedia.js` 活躍引用
