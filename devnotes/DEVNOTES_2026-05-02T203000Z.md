# DEVNOTES_2026-05-02T203000Z

## 背景

在 `realtime.ts` 與 `legacy.ts` 都移入 `services/display/*` 後，`function/runtime/dom.ts` 成為 `function/runtime` 內最後一個明確的 bootstrap owner。

它負責：

- 綁定 legacy DOM 事件
- 初始化 canvas/video/runtime state
- 啟動 realtime、camera、draw loop

這些責任屬於 display bootstrap / orchestration，而不是底層 runtime kernel。

## 本輪決策

### A. 將 bootstrap owner 移入 `services/display/runtime.ts`

`src/lib/services/display/runtime.ts` 不再只是薄 re-export，而是直接成為 display runtime bootstrap 的正式 owner。

### B. 刪除舊 `function/runtime/dom.ts`

完成改線後，刪除 `src/lib/function/runtime/dom.ts`。

### C. 保留 `function/runtime` 的 kernel 檔案

本輪不動的 `function/runtime/*` 目標是保留：

- `core.ts`
- `draw.ts`
- `gesture.ts`
- `particle.ts`
- `sprite.ts`
- `camera.ts`
- `mediapipe.ts`
- `cam.toggle.ts`

## 本輪不做

本輪刻意不做以下事項：

- 不改 runtime core/camera/particle 邏輯
- 不改 component 與 DOM 結構
- 不把 `camera.ts` 與 `mediapipe.ts` 再往上搬

## 驗證方式

本輪完成後需確認：

- `npm run check` 通過
- workspace 中不再有 `function/runtime/dom` import
- `services/display/runtime.ts` 成為正式 bootstrap owner

## 下一步

若本輪穩定，下一步應優先盤點：

1. `function/runtime/camera.ts` 是否可拆成 service owner + kernel helper
2. `function/runtime/mediapipe.ts` 是否仍屬 kernel，或應併入 camera service
3. 是否要把剩餘 `function/runtime/*` 重新命名為更明確的 kernel 目錄
