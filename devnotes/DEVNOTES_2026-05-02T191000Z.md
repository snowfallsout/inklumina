# DEVNOTES_2026-05-02T191000Z

## 背景

在上一輪刪除 `function/session/*` 之後，下一個最小且可驗證的 legacy 刪除點是：

- `src/lib/function/runtime/index.ts`

這個檔案目前只做：

- `export { bindDisplayRuntime, initDisplayRuntime } from './dom';`

但 workspace 內的正式 runtime 入口已經是：

- `src/lib/services/display/runtime.ts`
- `src/lib/services/displayRuntime.ts`（兼容 re-export）

因此 `function/runtime/index.ts` 已經成為零引用 barrel。

## 本輪決策

### A. 刪除 `function/runtime/index.ts`

刪除理由：

1. 無任何活躍 import
2. 有新的正式 owner/entrypoint
3. 保留它只會讓 `function/runtime` 看起來仍是公開 API 面

### B. 保留 `function/runtime/dom.ts`

雖然 barrel 可刪，但 `dom.ts` 仍由 `services/display/runtime.ts` 承接，因此本輪不動 `dom.ts` 與其他 runtime 內核檔案。

## 驗證方式

本輪完成後需確認：

- `npm run check` 通過
- workspace 中無任何對 `function/runtime/index.ts` 的依賴
- display runtime 仍從 `services/display/runtime.ts` 對外暴露

## 下一步

若本輪穩定，下一步應優先盤點：

1. `function/runtime/types.ts` 是否應被 `services/display/types.ts` 吸收或改為 shared type owner
2. `function/runtime/dom.ts` 中哪些 DOM 行為應進一步內聚到 component/state/service
3. `function/runtime/legacy.ts` 是否已縮到可拆除的程度
