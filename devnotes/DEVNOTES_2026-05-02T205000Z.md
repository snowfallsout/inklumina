# DEVNOTES_2026-05-02T205000Z

## 背景

在 `camera.ts` 移入 `services/display/camera.ts` 之後，`function/runtime` 還殘留兩個只被 camera service 使用的檔案：

- `src/lib/function/runtime/mediapipe.ts`
- `src/lib/function/runtime/cam.toggle.ts`

它們不再被任何 runtime kernel 直接共享，只作為 camera service 的實作輔助。

## 本輪決策

### A. 將 camera 相鄰 helper 收編到 `services/display/*`

本輪新增：

- `src/lib/services/display/mediapipe.ts`
- `src/lib/services/display/cam.toggle.ts`

讓 camera owner 與其專屬 helper 位於同一個 service 層。

### B. 刪除舊 `function/runtime` helper

完成改線後，刪除：

- `src/lib/function/runtime/mediapipe.ts`
- `src/lib/function/runtime/cam.toggle.ts`

## 本輪不做

本輪刻意不做以下事項：

- 不修改 MediaPipe handler 內容
- 不修改 camera badge 文案或 UI 行為
- 不改動 core/draw/gesture/particle/sprite 這些 kernel

## 驗證方式

本輪完成後需確認：

- `npm run check` 通過
- workspace 中不再有 `function/runtime/mediapipe` 與 `function/runtime/cam.toggle` import
- `function/runtime` 只剩真正 runtime kernel 檔案

## 下一步

若本輪穩定，下一步應優先盤點：

1. `function/runtime` 是否已可視為 kernel-only 區域
2. 是否要將 `function/` 目錄下剩餘 legacy JS 再分出 protected reference 與 active code
3. 是否要把 `function/runtime` 重新命名成更語義化的 kernel 目錄
