# DEVNOTES_2026-05-02T214000Z

## 背景

目前 `services` 底下還有幾個會讓維護面變得分散的殘留：

- `services/display/mediapipe.ts` 只被 `services/display/camera.ts` 使用
- `services/display/cam.toggle.ts` 只被 `services/display/camera.ts` 使用
- `services/core/particleEngine.ts` 只被 `components/display/Canvas.svelte` 使用
- `services/core/spriteCache.ts` 只被 `services/display/sprite.ts` 使用
- `services/api.ts` 是空白檔案且沒有任何引用

## 本輪決策

### A. 將 camera 私有 helper 內聯回 `services/display/camera.ts`

`display/mediapipe.ts` 與 `display/cam.toggle.ts` 的內容直接回收進 `display/camera.ts`，避免為單一 owner 再拆額外檔案。

### B. 移除 `services/core` 的單一消費者相容層

- `particleEngine.ts` 移到 `services/display/particleEngine.ts`
- `spriteCache.ts` 取消，`services/display/sprite.ts` 直接對接 `utils/sprites`

### C. 刪除空白與冗餘檔案

本輪刪除：

- `services/api.ts`
- `services/display/mediapipe.ts`
- `services/display/cam.toggle.ts`
- `services/core/spriteCache.ts`
- `services/core/particleEngine.ts`

並同步移除 `app.d.ts` 中對 `$lib/services/mediapipe` 的冗餘 `declare module`。

## 驗證方式

本輪完成後需確認：

- `npm run check` 通過
- workspace 中不再有 `services/core/*` 與空白 `services/api.ts`
- `display/camera.ts` 可單獨承接 camera 行為與其私有 helper
