# DEVNOTES_2026-05-02T202000Z

## 背景

在移出 runtime type owner 與 legacy bridge owner 之後，`function/runtime` 還有一個明顯不是核心演算法的檔案：

- `src/lib/function/runtime/realtime.ts`

它負責的內容主要是：

- display socket 綁定
- socket payload 對 display state 的同步
- session reset 時對 runtime 狀態的重置

這些職責更接近 display service orchestration，而不是底層 runtime kernel。

## 本輪決策

### A. 將 realtime owner 移入 `services/display/realtime.ts`

本輪新增：

- `src/lib/services/display/realtime.ts`

作為 display realtime/socket 綁定的正式 owner。

### B. `function/runtime/dom.ts` 改吃 service owner

`dom.ts` 仍然保留 runtime bootstrap 責任，但它改成從 `services/display/realtime.ts` 取得 realtime 綁定能力。

### C. 刪除舊 `function/runtime/realtime.ts`

完成改線後，刪除舊檔。

## 本輪不做

本輪刻意不做以下事項：

- 不改 `function/runtime/dom.ts` 的 DOM 初始化流程
- 不改 `function/runtime/core.ts` 粒子/畫面更新邏輯
- 不改 socket payload contract

## 驗證方式

本輪完成後需確認：

- `npm run check` 通過
- workspace 中不再有 `function/runtime/realtime` import
- realtime/socket binding 正式由 `services/display/realtime.ts` 持有

## 下一步

若本輪穩定，下一步應優先盤點：

1. `function/runtime/dom.ts` 是否可拆成 `services/display/bootstrap.ts` + 更薄的 runtime glue
2. `function/runtime/camera.ts` 是否可移入 `services/display/camera.ts`
3. `function/runtime` 是否只剩真正 kernel：`core/draw/gesture/particle/sprite/camera/mediapipe`
