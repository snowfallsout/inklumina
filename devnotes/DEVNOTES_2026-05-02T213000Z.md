# DEVNOTES_2026-05-02T213000Z

## 背景

上一輪雖然已將 `function/runtime` 重新命名為更直接的 kernel 路徑，但新的 `services/display/kernel/*` 又在 `services/display` 下多包了一層。

## 本輪決策

### A. 扁平化 `services/display/kernel/*`

本輪把以下檔案直接移回 `services/display/*`：

- `core.ts`
- `draw.ts`
- `gesture.ts`
- `particle.ts`
- `sprite.ts`

### B. 移除額外的 kernel 子目錄說明檔

`services/display/kernel/README.md` 一併刪除，避免在目前規模下再維護一層只服務於子目錄存在本身的文件。

## 本輪不做

本輪刻意不做以下事項：

- 不變更 kernel 行為
- 不處理 `display/mediapipe.ts` 與 `services/mediapipe.ts` 的重疊問題
- 不處理 `services/core/*` 合併問題

## 驗證方式

本輪完成後需確認：

- `npm run check` 通過
- workspace 中不再有 `services/display/kernel/*` 的 active import
- `services/display/*` 維持單層結構
