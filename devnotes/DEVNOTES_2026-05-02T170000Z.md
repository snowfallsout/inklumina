# DEVNOTES_2026-05-02T170000Z

## 背景

在前一輪修正後，專案曾短暫嘗試以 `src/lib/display/*` 作為 display owner。

但目前新的架構共識已經更清楚：

- `components`：UI / Svelte component
- `config`：global config settings
- `server`：server-only / SSR-only 邏輯
- `shared`：contract 與跨端共用型別/常量
- `state`：Svelte 5 runes state
- `services`：其餘不適合進入上述層級的 orchestration / adapter / domain service

在此共識下，display 與 mobile 的 domain service 應收斂到：

- `services/display/*`
- `services/mobile/*`
- `services/core/*` 僅保留少量核心服務層

同時，真正跨端共用的常量不再留在模糊的 `$lib/constants/*`，而應收斂到 `shared/constants/*`。

## 本輪決策

### A. `services/display/*` 作為正式 owner

本輪不再擴張 `src/lib/display/*`。

正式 owner 改為：

- `services/display/runtime.ts`
- `services/display/session.ts`
- `services/display/types.ts`

而 `services/displayRuntime.ts` / `services/displaySession.ts` 僅保留為過渡期兼容入口。

### B. `shared/constants/*` 作為 shared 常量入口

本輪開始將專案內仍殘留的 `$lib/constants/mbti`、`$lib/constants/vision` 引用統一切換到：

- `shared/constants/mbti.ts`
- `shared/constants/vision.ts`

這一步的目的是把「跨 display / mobile / server 的靜態資料」正式收斂到 shared 層。

### C. 先做入口與 import 收斂，不大搬 runtime 實作

本輪刻意只做：

- 目錄 owner 對齊
- 註解/檔頭說明對齊
- shared 常量 import 統一

本輪不做：

- `utils/state.ts` → `state/display.svelte.ts`
- `function/runtime/*` 大規模搬移
- `services/core/*` 與 `utils/*` compatibility shell 清理

## 本輪實作清單

1. 建立本文件，記錄新架構共識與本輪邊界
2. 更新 `DEVNOTES.md` 指向新的 snapshot

5. 執行 `npm run check` 驗證路徑與型別未被破壞

本輪完成後應確認：

- `npm run check` 通過
- repo 內不再依賴 `$lib/constants/mbti` 或 `$lib/constants/vision`
- `services/display/*` 的 owner 註解與實際路徑一致
