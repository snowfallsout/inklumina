# DEVNOTES_2026-05-02T190000Z

## 背景

在前幾輪重構之後，display session 的正式 owner 已經收斂到：

- `src/lib/services/display/session.ts`
- `src/lib/services/displaySession.ts`（兼容入口）

同時，display state owner 也已經從 `utils/state.ts` 轉移到 `state/display.svelte.ts`。

在這個前提下，舊的 `src/lib/function/session/*` 已不再承擔任何活躍路徑：

- workspace 內已無任何 `function/session` import
- 其內容僅剩舊版 client/panel/qr/utils/index 殼
- 這組檔案與目前 `services/display/session.ts` 形成雙軌維護風險

## 本輪決策

### A. 刪除整組 `function/session/*`

本輪刪除：

- `src/lib/function/session/client.ts`
- `src/lib/function/session/panel.ts`
- `src/lib/function/session/qr.ts`
- `src/lib/function/session/utils.ts`
- `src/lib/function/session/index.ts`

這一輪的刪除標準是：

1. 已無任何活躍 import
2. 已有新的正式 owner
3. 刪除後不需立即補新的 compatibility shell

### B. 暫不刪除 `function/runtime/*`

雖然 `function/session/*` 已經可以刪除，但 `function/runtime/*` 目前仍是 display runtime 內核的實際實作位置，因此本輪不動它。

## 本輪不做

本輪刻意不做以下事項：

- 不刪除 `function/runtime/*`
- 不整理空目錄
- 不重寫 `services/display/session.ts`
- 不處理 `function/session/*` 以外的 legacy shell

## 驗證方式

本輪完成後需確認：

- `npm run check` 通過
- workspace 中不再有 `function/session` 的 import
- display session 行為仍由 `services/display/session.ts` 承擔

## 下一步

若本輪穩定，下一步應優先檢查：

1. `function/runtime/index.ts` 是否已成為可刪除的空 barrel
2. `function/runtime/*` 中哪些檔案已可由 `services/display/*` 或 `services/core/*` 取代
3. `utils/*` 中是否還有與 runtime / display service 重複的 legacy 實作
