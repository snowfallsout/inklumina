# DEVNOTES_2026-05-02T160000Z

## 背景

上一輪 `DEVNOTES_2026-05-02T150000Z` 已經確立：`function/session` 與 `function/runtime` 不應再作為長期 public surface。

但目前實際程式碼仍存在兩個過渡問題：

1. `services/displayRuntime.ts` 仍直接指向 `function/runtime/dom.ts`
2. `services/displaySession.ts` 已經是 display session 的事實 owner，卻仍掛在 `services`，使 `services` 與 display domain 的責任混淆

因此本輪的目的不是再拆更多技術資料夾，而是先建立一個新的明確 owner：`src/lib/display/*`。

## 本輪決策

### A. 建立 display domain owner

新增：

- `src/lib/display/runtime.ts`
- `src/lib/display/session.ts`
- `src/lib/display/types.ts`

這三個檔案的定位如下：

- `display/runtime.ts`：display runtime bootstrap 的正式入口
- `display/session.ts`：display session panel / QR / REST orchestration 的正式入口
- `display/types.ts`：display runtime 相關型別的正式入口

### B. `services/*` 退回薄轉接層

本輪將：

- `services/displayRuntime.ts` 改為 re-export `display/runtime.ts`
- `services/displaySession.ts` 改為 re-export `display/session.ts`

這樣做的目的，是在不破壞既有 import path 的前提下，先把 owner 從 `services` 轉移到 `display`。

### C. 舊 runtime 開始改吃新 owner

`function/runtime/dom.ts` 的 session orchestration import，改為直接依賴 `display/session.ts`，而不是再走 `services/displaySession.ts`。

這一步的意義不是要立即刪除 `function/runtime`，而是讓舊實作開始跟隨新 owner。

## 本輪不做

本輪刻意不做以下事項：

- 不搬移 `function/runtime/core.ts`、`camera.ts`、`gesture.ts`、`particle.ts` 的實作位置
- 不處理 `utils/state.ts` → `state/display.svelte.ts`
- 不刪除 `function/session/*` 或 `function/runtime/*`
- 不清理 `utils/particle.ts`、`utils/pool.ts`、`utils/sprites.ts` 這些重複 engine 檔案

## 第一輪實作清單

1. 新增 `src/lib/display/runtime.ts` 作為 display runtime owner
2. 新增 `src/lib/display/session.ts` 作為 display session owner
3. 新增 `src/lib/display/types.ts` 作為 display runtime types owner
4. 將 `services/displayRuntime.ts` 改成薄轉接
5. 將 `services/displaySession.ts` 改成薄轉接
6. 將 `function/runtime/dom.ts` 改為依賴 `display/session.ts`

## 驗證方式

本輪完成後需確認：

- `npm run check` 通過
- `services/displayRuntime.ts` 不再直接指向 `function/runtime/dom.ts`
- `services/displaySession.ts` 不再承擔實作 owner，而只作薄轉接
- `function/runtime/dom.ts` 對 session orchestration 的依賴已切到 `display/session.ts`

## 下一步

若本輪穩定，下一輪應開始處理真正的 runtime engine owner 遷移：

1. `function/runtime/core.ts` → `display/runtime.ts` 聚合 owner

3. `utils/state.ts` → `state/display.svelte.ts`
